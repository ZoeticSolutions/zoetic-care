# zoetic.care/.well-known — domain association & callback files

These files are served over HTTPS from the apex domain so first-party apps and planned integrations
can bind to `zoetic.care`. GitHub Pages serves them as-is (a `../.nojekyll` file disables Jekyll so
this dot-directory isn't skipped).

## Present now

- **`apple-app-site-association`** (no extension, `application/json`)
  - `webcredentials` → **iOS passkeys** (WebAuthn RP = `zoetic.care`) for `V46GH2Q974.com.zoetic.care`.
  - `applinks` → **Universal Links**: `/invite/*`, `/portal/*`, `/app/*` open in the app (Patient
    Portal claim links, magic links).
- **`assetlinks.json`** — Android Digital Asset Links: `get_login_creds` (**Android passkeys /
  Credential Manager**) + `handle_all_urls` (**App Links**) for `com.zoetic.care`.
  - ⚠️ `sha256_cert_fingerprints` is a **placeholder** — replace with the real Android app-signing
    SHA-256 (Play Console → App integrity, or `keytool -list -v -keystore …`).

## To add when those integrations are set up

- **Sign in with Apple** (web/email relay): `apple-developer-domain-association.txt` — content is
  generated by Apple Developer → Identifiers → Services ID → domain registration. Don't fake it.
- **OAuth (Microsoft / Google / Apple) redirect URIs** are configured in each provider's console
  (e.g. `https://api.zoetic.care/auth/callback/<provider>`), **not** served from here.
- **`security.txt`** (RFC 9116): add once a monitored `security@zoetic.care` (or equivalent) exists.
